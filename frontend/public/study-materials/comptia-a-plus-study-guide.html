1. Mobile Devices

1.1 Laptop Hardware Components

  • Display types: TN (fast, poor angles), IPS (best color/angles), VA (high contrast), OLED (perfect blacks, power efficient)
  • Inverter: Powers CCFL backlights (older laptops); LED backlights do NOT use inverters
  • Digitizer: Touch-sensitive layer over display; separate from LCD panel
  • Webcam: Integrated above display; connects via internal USB or proprietary connector
  • Keyboard: Connects via ribbon cable to motherboard; function keys (Fn) toggle secondary functions
  • Touchpad: Capacitive surface; drivers control gestures; can be disabled via Fn key
  • Battery: Li-Ion or Li-Polymer; measured in mAh or Wh; calibrate by full discharge/charge cycle
  • DC jack: Power input connector; common failure point from physical stress
  • RAM: SO-DIMM form factor (laptop); DDR4 or DDR5; some soldered (not upgradeable)
  • Storage: 2.5" SATA SSD/HDD or M.2 NVMe/SATA SSD
  • Wireless card: Mini PCIe or M.2 (A/E key); 802.11 Wi-Fi + Bluetooth combo cards common
  • Speakers: Connected via audio header; stereo pair typically under keyboard or at base
  • Optical drive: Slim SATA; increasingly rare; external USB alternatives common
  • CPU: Soldered BGA package in most modern laptops; not user-replaceable
  • Thermal paste: Between CPU/GPU die and heatsink; dries out over time causing overheating

1.2 Laptop Display Components

  • LCD panel: Liquid crystal display; requires backlight (LED or CCFL)
  • LED backlight: Edge-lit or direct-lit; more efficient than CCFL; no inverter needed
  • CCFL backlight: Cold cathode fluorescent lamp; older technology; requires inverter
  • Backlight failure: Screen appears very dim; use flashlight test to confirm image still present
  • Broken screen: Cracks, dead pixels, lines; replace entire LCD assembly
  • Video cable: Connects motherboard GPU to display; runs through hinge; can fray
  • Wi-Fi antenna: Routed around display bezel for maximum signal; connects to wireless card

1.4 Mobile Device Synchronization

  • iCloud: Apple cloud sync; contacts, calendar, photos, iMessage, iCloud Drive
  • Google Account: Android sync; Gmail, Contacts, Calendar, Drive, Photos
  • Microsoft Exchange: Enterprise email/calendar/contacts; ActiveSync protocol

2. Networking

2.1 TCP/IP Basics

  • IPv4: 32-bit address; dotted decimal notation (e.g., 192.168.1.1); ~4.3 billion addresses
  • IPv6: 128-bit address; hexadecimal colon notation; 340 undecillion addresses
  • Subnet mask: Defines network vs. host portion; /24 = 255.255.255.0
  • Default gateway: Router IP; used to reach other networks
  • DNS server: Resolves hostnames to IP addresses
  • DHCP: Automatically assigns IP, subnet mask, gateway, DNS to clients
  • APIPA: 169.254.x.x; self-assigned when DHCP fails; indicates connectivity problem
  • Loopback: 127.0.0.1 (IPv4) or ::1 (IPv6); tests local TCP/IP stack

2.4 Network Hardware

  • Hub: Layer 1; broadcasts to all ports; creates collision domain; obsolete
  • Switch: Layer 2; forwards based on MAC address table; creates separate collision domains
  • Router: Layer 3; routes between networks using IP addresses; connects LAN to WAN
  • Wireless Access Point (WAP): Extends wired network wirelessly; connects to switch via Ethernet
  • Modem: Modulates/demodulates signal; converts digital to analog (DSL/cable)
  • Firewall: Filters traffic based on rules; hardware or software
  • PoE switch: Power over Ethernet; delivers power via Cat cable to WAPs, IP cameras, phones
  • Patch panel: Organizes cable runs in rack; connects to switch via patch cables
  • Managed switch: Configurable VLANs, QoS, port mirroring, SNMP monitoring
  • Unmanaged switch: Plug-and-play; no configuration; home/small office use

2.7 Network Connector Types

  • RJ-45: 8-pin; Ethernet (Cat 5e/6/6a); T568A or T568B wiring standard
  • RJ-11: 6-pin (4 used); telephone/DSL
  • LC: Small form-factor fiber; push-pull latch; common in data centers
  • SC: Square fiber connector; push-pull; older installations
  • ST: Bayonet fiber connector; twist-lock; older multimode
  • F-type: Coaxial; screw-on; cable TV/satellite
  • BNC: Coaxial; bayonet; older 10Base2 Ethernet, video
T568B is the most common wiring standard in the US. Straight-through cables (both ends T568B) connect different device types (PC to switch). Crossover cables (one end T568A, one T568B) connect same device types (switch to switch) ‚Äî though modern switches use Auto-MDIX to handle this automatically.

3.6 Power Supply Units (PSU)

  • Form factors: ATX (standard), SFX (small form factor), TFX (thin form factor)
  • 80 Plus ratings: 80 Plus (80%), Bronze (82%), Silver (85%), Gold (87%), Platinum (90%), Titanium (92%) efficiency

3.7 Cooling Solutions

  • Air cooling: Heatsink + fan; thermal paste required; tower coolers for high TDP CPUs
  • AIO liquid cooling: All-in-one; pump, radiator, fans; 120mm, 240mm, 360mm radiators
  • Custom loop: Separate pump, reservoir, radiator, water blocks; maximum cooling
  • Thermal paste: Fills microscopic gaps between CPU and heatsink; replace every 3‚Äì5 years
  • Case fans: Intake (front/bottom) and exhaust (rear/top) for positive/negative pressure airflow
  • Fan headers: 3-pin (voltage control) or 4-pin PWM (pulse width modulation, quieter)

Laser Printer Process (in order):

  1. Processing: RIP (Raster Image Processor) converts data to bitmap
  2. Charging: Primary corona wire/charge roller applies uniform negative charge to drum
  3. Exposing: Laser beam neutralizes charge in image areas on drum
  4. Developing: Toner (negatively charged) attracted to neutralized (positive) areas on drum
  5. Transferring: Transfer corona wire/belt transfers toner from drum to paper
  6. Fusing: Fuser assembly (heat + pressure) melts toner permanently onto paper
  7. Cleaning: Cleaning blade removes residual toner from drum
Laser printer steps mnemonic: "Please Create Every Document Through Fusing Carefully" ‚Äî Processing, Charging, Exposing, Developing, Transferring, Fusing, Cleaning

4. Virtualization & Cloud Computing

4.1 Virtualization Concepts

  • Hypervisor Type 1 (bare-metal): Runs directly on hardware; no host OS; VMware ESXi, Microsoft Hyper-V, Xen, KVM
  • Hypervisor Type 2 (hosted): Runs on top of host OS; VMware Workstation, VirtualBox, Parallels
  • VM (Virtual Machine): Complete virtualized computer with virtual CPU, RAM, storage, NIC
  • Snapshot: Point-in-time copy of VM state; allows rollback; not a backup
  • Template: Master VM image for rapid deployment of identical VMs
  • vCPU: Virtual CPU; mapped to physical CPU cores/threads
  • vRAM: Virtual RAM; allocated from host physical RAM
  • Virtual NIC: Emulated network adapter; connects to virtual switch
  • Virtual switch: Software switch connecting VMs to each other and physical network
  • Resource pooling: Multiple VMs share physical resources dynamically
  • Live migration: Move running VM between hosts without downtime (vMotion, Live Migration)

4.2 Cloud Service Models

ModelFull NameCustomer ManagesExamples

4.3 Cloud Deployment Models

  • Public cloud: Shared infrastructure; owned by provider (AWS, Azure, GCP); pay-per-use
  • Private cloud: Dedicated infrastructure; owned/operated by organization; on-premises or hosted
  • Hybrid cloud: Combination of public and private; data/apps can move between them
  • Community cloud: Shared by organizations with common interests (government, healthcare)

4.4 Cloud Characteristics (NIST Definition)

  • On-demand self-service: Provision resources without human interaction with provider
  • Broad network access: Available over network via standard mechanisms
  • Resource pooling: Provider resources pooled to serve multiple consumers
  • Rapid elasticity: Scale up/down quickly; appears unlimited to consumer
  • Measured service: Usage monitored, controlled, reported; pay for what you use

5. Hardware & Network Troubleshooting

5.1 Troubleshooting Methodology (CompTIA 7-Step)

  1. Identify the problem: Gather information, question users, identify symptoms, duplicate if possible, check recent changes
  2. Establish a theory of probable cause: Consider obvious causes first; question the obvious
  3. Test the theory: Confirm or deny theory; if confirmed, determine next steps; if not confirmed, establish new theory
  4. Establish a plan of action: Resolve the problem; identify potential effects
  5. Implement the solution: Apply fix; escalate if necessary
  6. Verify full system functionality: Confirm fix works; implement preventive measures
  7. Document findings: Record problem, cause, solution, and outcome

5.2 POST Codes & BIOS/UEFI

  • POST: Power-On Self-Test; hardware check during boot; beep codes indicate failures
  • BIOS: Basic Input/Output System; legacy firmware; MBR partition table; 16-bit; 1MB limit
  • UEFI: Unified Extensible Firmware Interface; modern replacement; GPT partition table; 64-bit; Secure Boot
  • Secure Boot: UEFI feature; verifies bootloader signature; prevents rootkits/bootkits
  • TPM: Trusted Platform Module; hardware security chip; stores encryption keys; required for Windows 11
  • CMOS battery: CR2032 coin cell; maintains BIOS settings and clock when powered off; replace if settings reset
  • Common beep codes (AMI BIOS): 1 beep = POST OK; 3 beeps = RAM error; 5 beeps = CPU error; 8 beeps = GPU error

7.3 Security Best Practices

  • Principle of least privilege: Grant minimum permissions needed to perform job function
  • MFA (Multi-Factor Authentication): Something you know + something you have + something you are
  • Strong passwords: 12+ characters; uppercase, lowercase, numbers, symbols; no dictionary words
  • Password manager: Securely store unique passwords for each account
  • Account lockout: Lock account after N failed login attempts; prevents brute force
  • Screen lock: Auto-lock after inactivity; require password to unlock
  • Encryption: BitLocker (Windows), FileVault (macOS), LUKS (Linux) for full-disk encryption
  • Patch management: Keep OS and applications updated; close known vulnerabilities
  • Antivirus/EDR: Real-time protection; scheduled scans; keep definitions updated
  • Firewall: Block unauthorized inbound/outbound connections
  • VPN: Encrypt traffic on untrusted networks (public Wi-Fi)
  • Data destruction: Shred documents; wipe drives (DoD 5220.22-M, Gutmann); degauss magnetic media; physically destroy SSDs

7.4 Windows Security Features

  • Windows Defender: Built-in antivirus/antimalware; real-time protection
  • Windows Firewall: Built-in host-based firewall; inbound/outbound rules
  • BitLocker: Full-disk encryption; requires TPM; recovery key backup essential
  • EFS (Encrypting File System): File/folder level encryption; NTFS only; tied to user account
  • UAC (User Account Control): Prompts for elevation when admin rights needed; prevents silent installs
  • Windows Hello: Biometric authentication; fingerprint, face recognition, PIN
  • Secure Boot: UEFI feature; verifies bootloader; prevents boot-level malware
  • Windows Sandbox: Isolated desktop environment; test suspicious software safely

  • BSOD (Blue Screen of Death): Stop error; hardware/driver failure; note stop code (e.g., IRQL_NOT_LESS_OR_EQUAL, PAGE_FAULT_IN_NONPAGED_AREA)
  • Boot failure: "BOOTMGR is missing" ‚Äî repair with Windows installation media; bootrec /fixmbr, /fixboot, /rebuildbcd
  • Application crash: Check Event Viewer Application log; update or reinstall application
  • Slow performance: Check Task Manager for high CPU/RAM/disk usage; scan for malware; check startup programs
  • Missing DLL: Reinstall application; run sfc /scannow; download from trusted source
  • Profile corruption: Create new user profile; copy data; delete old profile

9. Operational Procedures

9.1 Documentation & Change Management

  • Network diagram: Visual map of network topology; physical and logical
  • Asset inventory: Hardware/software inventory; serial numbers, licenses, locations
  • Knowledge base: Documented solutions to common problems
  • Change management: Formal process for making changes; request, approval, testing, implementation, rollback plan
  • Incident report: Document security incidents; what happened, impact, response, lessons learned
  • SLA (Service Level Agreement): Defines expected service levels; response times, uptime guarantees

3-2-1 Backup Rule: Keep 3 copies of data, on 2 different media types, with 1 copy offsite. This protects against hardware failure, local disaster, and theft.

9.3 Environmental Controls

  • Temperature: Servers: 64‚Äì80¬∞F (18‚Äì27¬∞C); ASHRAE A1 class: 59‚Äì89.6¬∞F
  • Humidity: 40‚Äì60% relative humidity; too low = static; too high = condensation
  • UPS (Uninterruptible Power Supply): Battery backup; protects against power outages and surges
  • Surge protector: Protects against voltage spikes; MOV (Metal Oxide Varistor) absorbs surges
  • ESD (Electrostatic Discharge): Can damage components; use anti-static wrist strap, mat, bags
  • Fire suppression: Halon alternatives (FM-200, Novec 1230) for server rooms; water damages equipment
  • Cable management: Organize cables; improve airflow; reduce tripping hazards

9.4 Professionalism & Communication

  • Use proper language; avoid jargon with non-technical users
  • Listen actively; let users finish explaining before responding
  • Set realistic expectations; communicate timelines
  • Follow up to ensure satisfaction
  • Maintain confidentiality; do not share user data
  • Respect user privacy; do not access personal files unnecessarily
  • Document all work performed in ticketing system
